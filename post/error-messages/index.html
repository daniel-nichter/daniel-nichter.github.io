Error Messages

Sep 4, 2020
#software design #errors

A lot has been written about error message in software, and here’s my take on the subject because it comes up with nearly every new engineer I work with.

First, let’s be clear: error messages are important, therefore they require as much attention to detail as normal code. It’s true that they may never been seen if the software is reliable, but that’s not the point. The point is: they’re like the life vests (personal flotation devices) under the seats of an airplane. We hope never to need them, but when we do need them, they’re indispensable. We know that software isn’t perfect and errors will occur. That’s ok if the error messages help users (and developer) quickly find and fix the problem.
